# 📝 기술 블로그 글쓰기 가이드

> 가볍게 시작하는 기술 블로그 글쓰기 - 부담 없이 나의 지식을 기록하고 공유하는 즐거움

---

## 🎯 핵심 철학

기술 블로그는 어려운 논문이 아닙니다. 복잡한 이론보다는 **누구나 바로 실천할 수 있는 핵심 비법**에 집중하세요.

**핵심 원칙**:
- 📌 **경험에서 출발**: 내가 직접 겪은 것이 가장 좋은 글감
- 📌 **독자는 과거의 나**: 몰랐던 나에게 설명하듯 작성
- 📌 **쉽고 간결하게**: 복잡한 것을 쉽게 풀어내기
- 📌 **실용성 우선**: 이론보다 실습과 예제

---

## 1단계: 글쓰기 전 - '무엇을, 누구에게' 정하기

탄탄한 준비는 글쓰기를 훨씬 수월하게 만듭니다.

### 주제는 내 경험에서 찾기

가장 좋은 글감은 바로 **'나의 경험'**입니다:

✅ **좋은 글감 예시**:
- 최근에 해결한 까다로운 버그
- 새로 사용해 본 라이브러리 후기
- 프로젝트에서 배운 교훈
- 공부하며 정리한 개념
- 성능 최적화 경험
- 에러 디버깅 과정

**왜 경험이 중요한가?**
- 내가 이미 잘 알고 있어서 쓰기 편함
- 방금 배운 것이라 설명이 더 자세함
- 독창적인 관점과 인사이트 포함
- 실전 경험이 담겨 신뢰도 상승

### 독자는 '과거의 나'로 설정하기

**"누구를 위해 글을 써야 할까?"** 막막하다면:

👉 **이 기술을 몰랐던 '과거의 나'**를 독자로 설정하세요.

**효과**:
- 과거의 내가 무엇을 궁금해했는지 명확히 알고 있음
- 어떤 부분에서 어려움을 겪었는지 기억함
- 그 눈높이에 맞춰 설명 가능
- 자연스럽게 독자에게 필요한 글이 됨

**예시**:
```
❌ 나쁜 예: "전문가를 위한 고급 알고리즘"
   → 너무 모호하고 독자가 불명확

✅ 좋은 예: "2주 전의 나처럼 비동기 프로그래밍이 헷갈리는 사람을 위한 가이드"
   → 구체적이고 공감대 형성
```

### 뼈대만 간단히 세우기

글이 산으로 가지 않도록 간단한 구조를 잡으세요.

**추천 구조**:
```
문제 제기 → 해결 과정 → 결론
서론 → 본론 → 요약
배경 → 구현 → 회고
```

**예시**:
```markdown
## 서론
- 문제 상황 설명
- 왜 이 글을 쓰게 되었나?

## 본론
### 첫 번째 접근
- 시도한 방법
- 결과와 문제점

### 두 번째 접근 (해결책)
- 개선된 방법
- 코드 예제
- 결과

## 결론
- 핵심 요약
- 배운 점
- 다음 단계
```

## 2단계: 글 쓰는 중 - '쉽고, 보기 좋게' 만들기

독자가 편안하게 읽을 수 있도록 만드는 것이 핵심입니다.

### 비유와 예시 활용하기

복잡한 개념은 친숙한 대상에 빗대어 설명하세요.

**좋은 비유 예시**:

```markdown
❌ 나쁜 예:
"CNN은 컨볼루션 레이어를 통해 feature extraction을 수행합니다."

✅ 좋은 예:
"CNN은 마치 사진에서 중요한 패턴(모서리, 곡선 등)을 자동으로
찾아내는 필터처럼 작동합니다. 여러 장의 필터를 겹쳐서 점점
복잡한 특징을 인식하는 원리입니다."
```

**비유의 효과**:
- 추상적 개념 → 구체적 이미지
- 전문 용어 → 일상 언어
- 이해하기 어려움 → 직관적으로 파악

### 점진적으로 설명하기

**간단 → 복잡 순서**로 설명하세요:

1. **가장 단순한 개념 소개**
2. **간단한 예제로 직관 제공**
3. **점차 복잡성 추가**
4. **실전 응용 사례**

**예시**:
```markdown
## 비동기 프로그래밍이란?

### 1단계: 동기 vs 비동기 비유
카페에서 주문하는 것으로 비유해볼까요?
- 동기: 앞 사람 주문 완료될 때까지 기다림
- 비동기: 주문하고 진동벨 받아서 다른 일 하다가 벨 울리면 받으러 감

### 2단계: 간단한 코드
```python
# 동기
result = long_task()  # 10초 대기
print(result)

# 비동기
await long_task()  # 기다리는 동안 다른 작업 가능
```

### 3단계: 실전 예제
웹 크롤링에서 100개 페이지를 가져올 때...
```

### 짧게 끊어 읽기 쉽게 만들기

**온라인 읽기의 특징**:
- 긴 글은 피로감 유발
- 스크롤하며 훑어보는 경우 많음
- 모바일 환경도 고려

**개선 방법**:

✅ **문단 분리**: 3~4문장 단위로 나누기
```markdown
❌ 나쁜 예:
Python은 프로그래밍 언어입니다. 배우기 쉽고 강력합니다. 데이터 과학, 웹 개발,
자동화 등 다양한 분야에서 사용됩니다. 많은 라이브러리가 있어 생산성이 높습니다.

✅ 좋은 예:
Python은 프로그래밍 언어입니다. 배우기 쉽고 강력합니다.

데이터 과학, 웹 개발, 자동화 등 다양한 분야에서 사용됩니다.

많은 라이브러리가 있어 생산성이 높습니다.
```

✅ **소제목 활용**: 섹션을 명확히 구분
```markdown
## 주요 기능
### 1. 데이터 분석
### 2. 웹 개발
### 3. 자동화
```

✅ **글머리 기호**: 나열 내용은 bullet point
```markdown
Python의 장점:
- 배우기 쉬운 문법
- 풍부한 라이브러리
- 활발한 커뮤니티
```

### 코드는 하이라이팅 필수

**구문 강조(Syntax Highlighting)**는 기술 블로그의 기본입니다.

````markdown
❌ 나쁜 예:
```
def hello():
    print("Hello")
```

✅ 좋은 예:
```python
def hello():
    print("Hello")
```
````

**코드 작성 원칙**:

1. **간결성**: 핵심만 담기
```python
# ✅ 좋은 예
def calculate_average(numbers):
    return sum(numbers) / len(numbers)

# ❌ 나쁜 예: 너무 길고 복잡
def calculate_average_with_validation_and_logging(
    numbers, validate=True, log=True, logger=None
):
    # ... 50줄의 검증 및 로깅 코드
```

2. **주석**: 왜(Why) 설명하기
```python
# ✅ 좋은 예: 왜 이렇게 했는지 설명
# 성능을 위해 리스트 컴프리헨션 사용 (for 루프보다 2배 빠름)
squares = [x**2 for x in range(1000)]

# ❌ 나쁜 예: 코드가 하는 일 반복
# 제곱 계산
squares = [x**2 for x in range(1000)]
```

### 어려운 용어는 링크로 처리

모든 전문 용어를 설명하려 하지 마세요.

**전략**:
- 핵심 용어: 직접 설명
- 부가 용어: 공식 문서 링크

```markdown
✅ 좋은 예:
이 기능은 Python 3.10부터 도입된
[Structural Pattern Matching](https://peps.python.org/pep-0634/)을
활용합니다.

더 자세한 내용은 [공식 문서](https://docs.python.org/)를
참고하세요.
```

## 3단계: 글쓴 후 - '제목과 다듬기'에만 집중

마지막 단계가 독자 유입을 결정합니다.

### 제목에 가장 공들이기

**제목은 독자의 첫인상**을 결정합니다.

**좋은 제목의 요소**:

1. **구체적인 숫자**
```
❌ "Python 팁 모음"
✅ "Python 생산성을 2배 높이는 5가지 팁"
```

2. **대상 독자 명시**
```
❌ "React 가이드"
✅ "React 입문자를 위한 완벽 가이드"
```

3. **핵심 키워드 앞배치**
```
❌ "효율적인 코드를 Python으로 작성하는 방법"
✅ "Python 코드 최적화: 성능을 10배 높이는 5가지 기법"
```

4. **가치 제안**
```
❌ "FastAPI 사용법"
✅ "FastAPI로 10분 만에 REST API 개발하기 (코드 포함)"
```

**제목 공식**:
```
[대상 독자] + [핵심 키워드] + [구체적 숫자/방법] + [가치 제안]

예시:
"초보자를 위한 Docker 완벽 가이드: 5단계로 배우는 컨테이너화"
```

### 소리 내어 읽어보기

**가장 효과적인 검토 방법**:

1. 글 전체를 **소리 내어 읽기**
2. 어색한 문장 발견
3. 즉시 수정

**효과**:
- 오타 발견
- 어색한 표현 개선
- 흐름 확인
- 문장 길이 조정

### 최종 체크리스트

발행 전 확인:

**기술적 검증**:
- [ ] 카테고리 소문자로 작성
- [ ] 코드 블록 언어 지정
- [ ] 이미지 경로 확인
- [ ] 링크 정상 작동

**내용 품질**:
- [ ] 서론-본론-결론 구조
- [ ] 소리 내어 읽어보기
- [ ] 핵심 내용 요약 (결론)
- [ ] 다음 단계 제안

```markdown

각 글에 시리즈 링크 추가:
