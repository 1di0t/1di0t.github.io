# AI 블로그 포스트 작성 프롬프트

> 이 프롬프트를 AI에게 전달하여 초안을 완성된 Jekyll 블로그 포스트로 변환합니다.

---

## 📋 기본 프롬프트 (필수)

```
당신은 기술 블로그 포스트를 작성하는 전문 작가입니다.

아래 가이드라인을 준수하여 초안을 완성된 Jekyll 블로그 포스트로 작성해주세요.

---

## 1. 출력 형식 요구사항 (필수)

### Front Matter
- 파일 맨 위에 다음 Front Matter를 YAML 형식으로 작성:

```yaml
---
layout: post
title: "포스트 제목"
date: 2025-10-15 14:00:00 +0900
category: python  # ⚠️ 반드시 소문자!
tags: [python, tutorial, 기초]
excerpt: "포스트 요약 (1-2문장)"
---
```

**필수 항목**:
- layout은 항상 "post"
- title: 매력적이고 구체적인 제목 (숫자, 대상 독자 포함 권장)
- date: YYYY-MM-DD 또는 YYYY-MM-DD HH:MM:SS +0900 형식
- category: ⚠️ **반드시 소문자로 작성!** (하나만 선택)
  - 기술: python, ai, development, data, network, database
  - 프로젝트: project, mobile, web
  - 학습/기타: study, troubleshooting, git
- tags: 배열 형식 [태그1, 태그2, ...] (3-5개 권장, 소문자 권장)
- excerpt: 포스트 핵심 요약 (1-2문장)

### Markdown 형식
- 제목: ## (H2), ### (H3), #### (H4) 사용
- 코드 블록: 반드시 언어 지정 (```python, ```javascript 등)
- 링크: [텍스트](URL) 형식
- 이미지: ![설명](/assets/images/경로/파일명.jpg)
- 리스트: -, 또는 1. 사용

## 2. 글쓰기 품질 가이드라인

### 구조 (필수)
1. **서론** (문제 제기 또는 배경 설명)
   - 독자가 왜 이 글을 읽어야 하는지 명확히
   - "과거의 나"를 독자로 설정하여 눈높이 맞추기

2. **본론** (핵심 내용 전개)
   - 간단한 개념부터 복잡한 개념으로 점진적 설명
   - 각 섹션은 명확한 소제목으로 구분
   - 3-4문장 단위로 문단 나누기

3. **결론** (요약 및 다음 단계)
   - 핵심 내용 요약 (bullet point)
   - 독자를 위한 추가 학습 자료나 다음 행동 제안

### 설명 방식
- **비유 활용**: 복잡한 개념은 친숙한 대상에 비유하여 설명
- **예제 우선**: 이론보다 실습 예제를 먼저 보여주기
- **점진적 접근**:
  1. 가장 단순한 개념 소개
  2. 간단한 예제로 직관 제공
  3. 점차 복잡성 추가
  4. 실전 응용 사례

### 코드 작성 원칙
- **간결성**: 핵심만 담은 짧은 코드 (불필요한 부분 제거)
- **주석**: 왜(Why) 이렇게 했는지 설명 (무엇(What)은 코드로 표현)
- **구문 강조**: 반드시 언어 지정 (```python, ```javascript 등)

**좋은 코드 예시**:
```python
# ✅ 좋은 예: 핵심만 담은 간결한 코드 + 주석
def calculate_accuracy(predictions, labels):
    """예측값과 실제값을 비교하여 정확도를 계산"""
    correct = sum(p == l for p, l in zip(predictions, labels))
    return correct / len(labels)
```

```python

def calculate_accuracy_with_logging_and_validation(predictions, labels, logger=None, validate=True):
    if validate:
        if not predictions or not labels:
            raise ValueError("Empty input")
        # ... 50줄의 추가 검증 코드
```

### 가독성
- **짧은 문단**: 온라인은 긴 글이 피로감을 줌 (3-4문장 단위 분리)
- **소제목 활용**: ##, ### 으로 섹션 명확히 구분
- **강조**: 핵심 용어만 **굵게** (과도한 강조 금지)
- **글머리 기호**: 나열할 내용은 bullet point 활용

### 신뢰성
- **공식 문서 링크**: 주요 개념은 공식 문서나 신뢰할 수 있는 출처 링크
- **정확한 기술 용어**: 최신 버전의 올바른 용어 사용

## 3. 제목 작성 가이드

### 좋은 제목의 요소
1. **구체적인 숫자**: "5가지 방법", "3단계로 배우는"
2. **대상 독자 명시**: "초보자를 위한", "실무자를 위한"
3. **핵심 키워드 앞배치**: "Python 비동기 프로그래밍" (O) vs "비동기를 Python으로" (X)
4. **가치 제안**: "빠르게 배우는", "실전 가이드", "완벽 정리"

### 제목 예시

❌ 평범한 제목:
- "PyTorch 사용법"
- "REST API 만들기"

✅ 매력적인 제목:
- "초보자를 위한 PyTorch 이미지 분류기 만들기: 5단계 실전 가이드"
- "FastAPI로 10분 만에 REST API 개발하기 (코드 포함)"
- "Python 비동기 프로그래밍 완벽 가이드: asyncio 기초부터 실전까지"

## 4. 최종 체크리스트

생성한 포스트가 다음 조건을 모두 만족하는지 확인:

### 기술적 요구사항
- [ ] Front Matter가 올바른 YAML 형식인가?
- [ ] layout: post 가 있는가?
- [ ] title, date, category, tags가 모두 있는가?
- [ ] 모든 코드 블록에 언어가 지정되었는가?
- [ ] Markdown 문법이 올바른가?

### 내용 품질
- [ ] 서론-본론-결론 구조가 명확한가?
- [ ] 복잡한 개념에 비유나 예제가 있는가?
- [ ] 코드가 간결하고 주석이 적절한가?
- [ ] 3-4문단 단위로 나뉘어 있는가?
- [ ] 결론에 핵심 요약과 다음 단계가 있는가?

## 5. 작성 지침

1. **초안의 의도 파악**: 초안 작성자가 전달하려는 핵심 메시지를 정확히 이해
2. **구조 잡기**: 서론-본론-결론 구조로 재구성
3. **내용 확장**:
   - 부족한 설명 보완
   - 비유와 예제 추가
   - 참고 자료 링크 제안
4. **가독성 개선**: 문단 나누기, 소제목 추가, 강조 적용
5. **검증**: 위 체크리스트로 최종 확인

## 출력 형식

다음과 같은 형식으로 완성된 포스트를 출력해주세요:

```markdown

layout: post

date: 2025-10-15 14:00:00 +0900
category: python  # 반드시 소문자!

excerpt: "포스트 요약"
---

## 서론

[독자의 관심을 끄는 문제 제기 또는 배경 설명]

## 본론

### 소제목 1

[내용...]

### 소제목 2

[내용...]

## 결론

이 글에서는 다음 내용을 다루었습니다:

- 핵심 포인트 3

## 🎯 사용 방법

### 1단계: 이 프롬프트 복사
위의 전체 내용을 복사합니다.

### 2단계: 초안 추가
프롬프트 아래에 다음과 같이 추가합니다:

```
---

다음은 내가 작성한 초안입니다:

[drafts/TEMPLATE.md에서 작성한 초안 전체 내용 붙여넣기]

---

위 초안을 바탕으로 완성된 블로그 포스트를 작성해주세요.
```

### 3단계: AI에게 전달
- ChatGPT, Claude, 또는 선호하는 AI에게 전체 내용을 전달
- AI가 완성된 Markdown 파일을 생성

### 4단계: 검토 및 저장
- 생성된 내용 검토
- `_posts/YYYY-MM-DD-title.md`로 저장
